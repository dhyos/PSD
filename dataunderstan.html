Understanding data#

Dalam kerangka kerja CRISP-DM (Cross Industry Standard Process for Data Mining), tahap Data Understanding merupakan langkah kedua setelah Business Understanding. Tujuannya adalah untuk memahami data yang dimiliki secara menyeluruh sebelum masuk ke tahap pemodelan

📊 Macam-macam Tipe Data#

Data dalam ilmu data dapat diklasifikasikan dari berbagai sudut pandang. Berikut adalah penjelasan tipe data ditinjau dari sifat, sumber, representasi, dan struktur:

1. Ditinjau dari Sifat Data#

  • Data Kualitatif (Qualitative Data)

    • Data berbentuk kategori atau deskripsi, tidak berupa angka.

    • Contoh: jenis kelamin (laki-laki/perempuan), warna mobil, merek motor.

  • Data Kuantitatif (Quantitative Data)

    • Data berbentuk angka, bisa dihitung dan diukur.

    • Dibagi menjadi:

      • Diskrit (Discrete) → berupa hitungan, tidak bisa pecahan. Contoh: jumlah unit terjual.

      • Kontinu (Continuous) → berupa ukuran, bisa dalam pecahan. Contoh: harga kendaraan, tinggi badan.

2. Ditinjau dari Sumber Data#

  • Data Primer

    • Data diperoleh langsung dari sumber pertama melalui survei, wawancara, observasi, atau eksperimen.

    • Contoh: kuesioner pelanggan PT. Hasjrat Abadi.

  • Data Sekunder

    • Data diperoleh dari pihak lain atau sumber yang sudah tersedia.

    • Contoh: laporan keuangan perusahaan, data BPS, dataset publik dari Kaggle.

3. Ditinjau dari Representasi Data#

  • Data Numerik

    • Direpresentasikan dalam bentuk angka.

    • Contoh: harga, kuantitas penjualan, umur.

  • Data Kategorikal

    • Direpresentasikan dalam bentuk kategori/label.

    • Contoh: tipe kendaraan (MPV, SUV, Hatchback).

  • Data Ordinal

    • Direpresentasikan sebagai kategori tetapi ada tingkatan.

    • Contoh: kepuasan pelanggan (sangat puas, puas, kurang puas).

  • Data Biner

    • Hanya memiliki dua nilai.

    • Contoh: Ya/Tidak, Aktif/Nonaktif.

  • Data Teks & Multimedia

    • Representasi dalam bentuk unstructured data.

    • Contoh: ulasan pelanggan (text), foto kendaraan (image), rekaman iklan (video).

4. Ditinjau dari Struktur Data#

  • Data Terstruktur (Structured Data)

    • Tersimpan dalam format tabel dengan baris dan kolom yang jelas.

    • Contoh: database SQL berisi data transaksi.

  • Data Semi Terstruktur (Semi-Structured Data)

    • Tidak seketat tabel, tetapi masih punya label/tag.

    • Contoh: file XML, JSON, log server.

  • Data Tidak Terstruktur (Unstructured Data)

    • Tidak punya format tertentu, biasanya data teks bebas atau media.

    • Contoh: komentar media sosial, gambar produk, video iklan.

Pengumpulan Data#

Dalam tahap ini kita harus mendapatkna data dari perusahaan PT. Hasjrat Abadi Lapulu bisa menggunakan data primer yaitu data yang kita peroleh langsung di lapangan atau data sekunder yang bisa kita dapatkan lewat media seperti https://www.kaggle.com/ dll. disini saya menggunakan data yang tersedia di kagle https://www.kaggle.com/datasets/raflyahmadzainur/dataset-pt-hasjrat-abadi-lapulu kita bisa mendowload data tersebut atau menarik datanya langsung dari kagle. merupakan proses memahami kualitas data dan memastikan bahwa data yang akan kita analisis atau gunakan dalam model machine learning sudah bersih, lengkap, dan sesuai format yang diharapkan. Hal yang perlu diperhatikan adalah:

Missing Values: Menemukan data yang hilang atau tidak lengkap.

Outliers: Mendeteksi nilai-nilai ekstrem yang mungkin merupakan kesalahan atau anomali.

Duplikasi Data: Mengidentifikasi baris atau entri yang duplikat.

Inkonsistensi: Memeriksa apakah ada ketidaksesuaian dalam format atau nilai data.
